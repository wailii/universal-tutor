# 🎓 universal-tutor

**通用 · 苏格拉底式 · 可持久化的学习导师**

把任何学科从「听得懂概念」逼到「能做判断、敢跟人对线、扛得住追问」。

一个 Skill，热插拔任意课程；跑在 Claude Code / Codex / 任意 Agent 上。

`零依赖（仅需 Python 3）` · `热插拔多学科` · `进度落文件，关掉再开无缝续上`

</div>

---

## 这不是「AI 陪你聊学习」

市面上的 AI 学习工具，本质是你问、它讲——讲得越顺，你越觉得懂了，其实没留下什么。这套系统反着来：

| | 普通 AI 陪学 | universal-tutor |
|---|---|---|
| 怎么教 | 你问它答，灌输 | **苏格拉底逼问**，把理解的重量压回你身上；卡住时降台阶，不直接给答案 |
| 算不算学会 | 你觉得懂了就行 | **必须过测验**（按认知层级出题），连续两次达标且跨天才算「掌握」 |
| 会不会忘 | 聊完就忘 | **间隔 + 交错复习**（SM-2 自动排期），到期把你拽回来重测 |
| 似懂非懂 | 发现不了 | **置信度校准**揪出「自信却答错」的危险区；**误概念账本**把搞混的点做成可清的待办 |
| 业界对标 | 凭记忆瞎说 | 涉及现状**一律联网核实**，严禁编造，逼出「谁在做、差异在哪」 |
| 学到一半关了 | 重开就断片 | 每天学的实时落文件，**跨天自动续上**、自动补当日小结 |

一句话：**逼你主动回忆、隔时再练、打乱来练、适度卡住、按层级验收、专打你以为懂其实不懂的地方。** 这就是它「立得住」的全部理由。

## 用起来长这样

你不用学任何命令，直接说人话：

```
你：学一下训推平台
导师：（先铺一句最小地基）训推平台把「数据→训练→推理」这条只有专家能跑通的链路标准化了。
      那我问你——如果不存在这个平台，算法工程师每天会卡在哪？平台到底替他们消灭了什么成本？
你：……大概是环境配置、调度这些杂活？
导师：对一半。你说的是「重复劳动」。再往深想一层：这些杂活里，哪一件如果做不好，
      会直接让一个训练任务「跑到一半全废」、而不只是慢一点？
你：checkpoint？断了能续？
导师：✅ 抓到关键了。那作为 PM，断点续训你要不要做成默认开启……
```

它会自己挑节点、铺地基、苏格拉底追问、联网对标、出题测验、记录进度——你只管动脑子答。

## 🚀 安装

> 需要 **Python 3**（标准库即可，零第三方依赖）。

**① 一条命令**（任意带 shell 的 Agent / 终端）
```bash
curl -fsSL https://raw.githubusercontent.com/wailii/universal-tutor/main/install.sh | bash
```
自动探测 skills 目录（Claude Code `~/.claude/skills/`、Codex `~/.codex/skills/`）→ 拉取 → 自检。

**② Claude Code 原生插件**
```
/plugin marketplace add wailii/universal-tutor
/plugin install universal-tutor@universal-tutor-mkt
```

**③ 一句提示词**（发给任意 Agent，它自己装）
```
帮我装 github.com/wailii/universal-tutor 这个学习 skill，
clone 到我的 skills 目录后跑一下自检，装好告诉我有哪些课、怎么开始。
```

> 🇨🇳 国内 `raw.githubusercontent.com` 可能慢——把仓库镜像到 Gitee，安装脚本里 `UT_REPO` 指过去即可（见 [INSTALL.md](INSTALL.md)）。没有网络也能装：用离线自解压器 `universal-tutor-offline-installer.sh`。

装完直接对你的 Agent 说「学训推平台」「考考我」「我想学 X」。

## 📚 内置两门课，也能现场建任意课

| 课程 | 节点 | 说明 |
|---|---|---|
| **大模型训推 + 算力调度** | 27 | 把 PM 带到「能做训推/调度平台的架构取舍、敢跟研发对线」 |
| **RAG 系统设计** | 53 | 用建包流水线**现场生成**的范例：每个节点目标具体、引用经联网核实全部真实 |

**没有你想学的？** 直接说「我想学 ⟨某主题⟩」，导师会走一条建包流水线**现场给你造一门**：
澄清你的目标 → 联网深调研领域骨架 → **逐节点独立深挖**（杜绝「越往后越敷衍」）→ 逐节点质检打回 → 查漏补缺 → 按你的水平裁剪。

> 想同时学好几门？都装着就行，一句话切换，进度/复习按课隔离——**不是装 N 个 Skill，是一个 Skill + N 张可插拔「卡带」**，永远不会触发词打架。

## 🧠 它怎么做到「立得住」

**两层架构**：

- **状态引擎**（`scripts/tutor.py`，零依赖）——负责所有确定性脏活：日期、进度、解锁、复习排期、置信度校准、误概念账本、落盘。**凡是它能干的，绝不靠模型自觉。**
- **苏格拉底引擎**（模型本身）——按 `references/` 里的教学协议提问、对标、测验、写日志。

**学科与引擎彻底解耦**：每门课 = `packs/<id>/` 一个目录（课程图 + 学习者画像 + 对标锚点）。加一门课不动一行引擎代码。

每个知识点都走同一套**学习循环**：入口校准 → 铺最小地基 → 苏格拉底深挖 → 联网对标 → 测验门禁 → 实时落盘 → 间隔+交错复习。背后每个机制都对应一条被验证的学习科学原理（检索练习 / 间隔 / 交错 / 合意困难 / Bloom 分层 / ZPD / 校准 / 误概念矫正 / 掌握学习 / 迁移）——详见 [`references/learning_framework.md`](references/learning_framework.md)。

## 🛠 命令速查

> **你基本不用记这些**——导师（AI）会在背后自动替你跑。下面是给想了解内部机制、或想手动看进度的人。

| 命令 | 作用 |
|---|---|
| `boot` | **开机校准**：每次开始学习时跑的第一件事。它告诉导师「现在该干嘛」——在学哪门课、是否跨天要补小结、有哪些到期复习/需重学/误概念、有哪些可学节点、当前进度。 |
| `init` | **初始化**：第一次给某门课建立进度档案。一门课只需一次；现在首次 `boot` 会自动 `init`，所以你基本碰不到它。 |
| `packs` / `use <id>` | 看装了哪些课 / 切换当前课 |
| `progress [--all]` | 看进度条（★已掌握 ✓已学 …学习中 ○可学 ·锁定） |
| `review [--all]` | 看今天到期复习 + 需重学 |
| `validate` / `audit` | 课程结构校验 / 深度体检（揪「敷衍」节点）——建课时用 |
| `master` / `log` / `miss` | 测验评分 / 落盘 / 误概念账本（导师自动调用） |

```bash
python scripts/tutor.py packs        # 看内置课
python scripts/tutor.py progress --all   # 看所有课进度
```

## ⚙️ 设计取舍（想改的话）

- **复习算法**：零依赖 SM-2 简化版（`tutor.py` 的 `schedule()`）。想换 Anki 同款 FSRS：`pip install fsrs` 替换该函数即可。
- **掌握判定**：测验 ≥4 分达标，连续两次且跨天 → 已掌握；mastered 到期仍会被抽查。
- **课程是单一真相源**：节点只定义在 `packs/<id>/curriculum.json` 一处，改课直接改它，引擎下次自动纳入、重算解锁。
- **课程 / 进度分离**：`packs/` 是不可变课程，`state/`（本地、不入库）是你的进度，互不污染。
- **教学内容不预写死**：课程图只有骨架和引子，真正的讲解每次现场生成 + 联网，保证新鲜、有出处、不幻觉。
- **跨 Agent**：优先 Python 宿主；无联网/无 Python 的降级路径见 [`AGENTS.md`](AGENTS.md)。

## 📂 目录结构

