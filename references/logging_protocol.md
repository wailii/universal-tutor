# 落盘与启动校准协议

用户的核心诉求：原汁原味记录每天学了什么，且系统能跨天、跨会话无缝续上。难点是模型在连续对话里没有可靠时间感、也分不清长对话里哪几轮属于哪天。解法是**实时落盘 + 启动校准**，把日期判断完全交给脚本（系统时间），不靠你的印象。

> 多包注意：日志和状态按包命名空间隔离——`logs/<pack>/YYYY-MM-DD.md`、`state/<pack>/`。所有命令默认作用于当前 active 包（`tutor.py use` 切换，或 `--pack` 指定）。

## 一、为什么是「实时落盘」而不是「事后切分」

不要等一天结束再回头把长对话切成日志——那条路最难走，你分不清哪轮属于哪天。
正确做法：**每讲透一个片段，立刻 `tutor.py log` 写当天文件**。日期由脚本取系统时间决定。于是：
- 昨天的内容昨天就写进了 `logs/<pack>/<昨天>.md`；
- 今天接着聊，自动写进 `logs/<pack>/<今天>.md`；
- 三天的学习天然变成三个文件，**根本不存在「从长对话中间切一段」这个动作**。

对话上下文只是你的工作记忆，真正的存储是这些按日期命名的文件。

## 二、启动校准协议（每次会话开场，硬性，不许跳过）

被唤起后第一件事跑 `python scripts/tutor.py boot`，它会做这些确定性的事并把结果给你：

1. **取今天日期**（系统时间，不接受任何人传日期）。
2. **报当前 active 包**（学多门课时先确认在学哪门；要换用 `use`）。
3. **跨天检测**：今天 ≠ 上次活跃日 → 报跨天，并指明要给哪个 `logs/<pack>/<上次日期>.md` 补当日小结。
4. **到期复习检测**：本包 + 其它包的到期项。
5. **开放误概念 + 校准危险区**：拎出来提示复习优先打。
6. **可学节点 + 当前 focus + 进度摘要**。

然后 boot 会（在没有待补小结时）把「上次活跃日」推进为今天。

## 三、boot 之后必须做的动作（按顺序）

1. **若报跨天** → 立刻给上一天的日志补**当日小结**（格式见第四节）。这是 lazy 结算：不指望用户正经宣布「今天学完了」，等下次开机检测到跨天时回头补。用户也可随时手动说「今天到这」让你当场补。
2. **若有到期复习 / 开放误概念** → 优先安排（连环追问 drill），清完再学新内容。
3. **读可学节点 + focus** → 跟用户确认今天学什么，开始学习循环。

## 四、当日小结格式（追加到当天日志末尾）

跨天补结或用户主动收尾时，在对应日志末尾追加：

```bash
cat <<'EOF' | python scripts/tutor.py log __DAILY_SUMMARY__
### 当日小结
- 今天学了：<节点ID + 一句话>
- 掌握度变化：<哪些升到 learned/mastered，哪些还卡着>
- 暴露/纠正的误概念：<对应 misconceptions 里的>
- 明天该复习：<review 列出的到期节点>
EOF
```

（`__DAILY_SUMMARY__` 只是标记节点名；它不在课程表里，不影响进度。引擎会把它写进"待补小结那天"的日志而非今天。）

## 五、落盘纪律

- **实时**：每个学透的片段当场 `log`，不要攒。攒着就会丢、会错位。
- **原汁原味**：日志记的是真实讲清楚的内容和结论（含对标出处链接），不是过度压缩的提纲——用户要的是能回看的原始记录。
- **缺口走 miss**：认知断层记 misconceptions，别混在日志正文里。
- **约束力弱的宿主**：你更容易聊嗨忘记落盘。把「开场先 boot」「每片段后 log」当成和苏格拉底纪律同级的硬动作，每轮自检里盯住。
