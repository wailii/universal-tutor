# 测验与掌握评估（让"掌握"立得住）

「掌握」不能由用户自评。一个节点学完，必须用本协议验证，再按口径打 1-5 分写回引擎（`tutor.py master <id> <分> --confidence <low|med|high> --bloom <层级>`）。**连续两次 ≥4 才算真掌握。**

这是相对原版最大的升级：测验不再是"会不会复述"，而是**按认知层级验收 + 校准 + 迁移**三件套。

---

## 一、入口校准（placement）：先别教，先探深浅

教一个新节点前，先用一两个开放问题探用户已有水平：
- 已经能讲清核心取舍 → 直接进测验，过了就 `master`、跳过讲解（按真实水平裁剪，省他时间）。
- 半懂 → 从他的断层处切入，地基只补缺的那块。
- 全新 → 走完整学习循环。

## 二、按 Bloom 层级出题（核心升级）

每个节点在 `curriculum.json` 里标了 `bloom`。**测验必须考到这个层级，低于它不算达标。** 对照表：

| 层级 | 这层在问什么 | 题目形态 |
|---|---|---|
| `understand` 理解 | 能用自己话讲清是什么/为什么 | "不看材料，用大白话讲：它是什么、为解决什么、核心取舍" |
| `apply` 应用 | 能把它用到一个具体案例 | "给你这么个场景，你会怎么用它/怎么配？为什么" |
| `analyze` 分析 | 能比较、拆解、讲取舍 | "它和 X 取舍的轴是什么？什么场景选哪个，为什么" |
| `evaluate` 评价 | 能为一个决策辩护/挑刺 | "有人主张这么做，你赞成吗？依据是什么？它的弱点在哪" |
| `create` 创造 | 能设计/组合出新方案 | "让你设计一个满足这些约束的方案，你怎么搭、取舍在哪" |

达到并答好某层，用 `--bloom <层级>` 记录，引擎会留住"达到过的最高层级"。

## 三、置信度校准（治"自信却错"）

打分**之前**，先让用户自报"你有多确定"（low/med/high），再核对答案，用 `--confidence` 记录。引擎会自动统计：
- **high 置信 + <4 分** → 进"校准危险区"，boot 会点名，复习优先重测。这是最危险的盲区（他以为懂，所以不会主动复习）。
- **low 置信 + ≥4 分** → 其实会但不自信，给正反馈，别让他低估自己。

## 四、测验形式（按节点难度和包配置选用，越往后越硬）

形式清单见 `pack.json` 的 `assessment_styles`；通用四类 + 实战情境：

1. **概念自测**：不看材料用大白话讲清。抓两件事：(a) 真懂还是背术语；(b) 表达是否结论先行、是否啰嗦发散。
2. **连环追问 drill**：就一个点不依不饶往下刨，直到答不上来。**目的不是难倒，是精确定位理解的断层**——断在哪，那里记 `miss`。检验真懂假懂最有效。
3. **迁移题**（升级新增）：把概念搬到一个**没讲过的新情境**让他用。近迁移（同领域换案例）→ 远迁移（跨领域类比）。真懂才迁得动。
4. **实战情境**（每包自定义，见 `assessment_styles`）：还原他真实会遇到的对抗场景，让他站住判断。例：
   - 训推/调度包：**模拟研发对线**（扮强势研发质疑决策）/**模拟面试追问**（强制 90 秒、结论先行，超时打断重讲）。
   - 其它包按 `assessment_styles` 配置（如模拟庭辩、模拟接诊、模拟答辩）。

## 五、评分口径（1-5，引擎达标线 4 分）

- **5｜真掌握**：用自己的话讲清、结论先行不啰嗦；能说出核心取舍；接得住连环追问到合理深度；达到该节点的 Bloom 层级；要对标的话能说出≥1 个具体差异点。
- **4｜达标**：核心讲对，取舍基本清楚；追问接住大部分；Bloom 层级基本够。（≥4 才计入连续达标）
- **3｜夹生**：术语会说但一追就露怯；取舍说不清或答反。→ 退回学习中，短期重排。
- **2｜表面**：只会复述定义，不理解动机；问为什么就卡。
- **1｜没学进去**：概念都没立住。

## 六、评分纪律（硬）

- **对错以真实出处为准**（见 `grounding_protocol.md`），不以你脑补的标准答案为准；拿不准先查再判。
- 打分前**先把判断依据简短讲给用户**（哪答好了、哪个断层、对标缺了啥），再给分——分数是结果，反馈才是价值。
- 任何测验里暴露的断层、口误、混淆，立刻 `tutor.py miss add`（见 `misconception_protocol.md`）。
- 复习到期的节点，直接用**连环追问 drill**（第 2 类）验证留存，再重新 `master` 评分。
- 掌握会遗忘：`mastered` 的节点到期仍会被引擎拎出来抽查，扛不住就退回——这不是 bug，是设计。
