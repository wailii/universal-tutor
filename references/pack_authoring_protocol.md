# 建包流水线（给任意新学科现场造一门"立得住、不敷衍"的课）

当用户想学一个**还没有内置包**的主题时，你切换成"建包师"角色，按本流水线给他**现场造一张高质量卡带**。

## 头号铁律：杜绝"越往后越敷衍"

LLM 一次性生成长列表时有个**已知通病**：前面几个节点用力、越往后越摆烂——objective 越写越短、种子问题越来越像送分题、对标越来越空，甚至把一大块硬骨头一句话带过、或者干脆砍单凑个整数。**这是这套系统最容易塌的地方，必须从机制上堵死，不能靠你自觉。**

机制就一条：**绝不把整张课程表一次性生成。** 改成"先铺广度骨架 → 每个节点各自独立深挖 → 逐节点质检打回 → 查漏补缺到挤不出新东西"。每个节点是一次**独立专注**的任务，第 1 个和第 30 个节点享受同样的深度预算。

> 产物固化：整条流水线只在建包时跑一次，结果写进 `packs/<id>/`。之后每天学就是读这张固化的卡带。

---

## 第 1 步 · 澄清对话（2~4 轮，动态，不是问卷）

把"我想学 X"收敛成一门为这个人定制的课。每轮根据上一轮回答决定下一个问题，问到：学什么+为什么学（定深度/评估风格/orientation）、现在的起点（定从哪切入、跳过什么）、能力终态（定 Bloom 目标）、时间节奏（定粒度）。产出 `learner_profile` + `goal` + `target_depth`，先写进 `pack.json`。

## 第 2 步 · 联网深调研（决定"扎不扎实"的命脉）

**不要凭训练记忆列大纲。** 复用环境里的 **`deep-research`** skill（或 WebSearch/WebFetch 自己扇出）把领域**真实的知识骨架**捞出来：权威教材/课程大纲、综述、经典论文/标准、一线实践。沉淀出"该有哪些核心概念、依赖先后、每块对标谁"。

## 第 3 步 · 只铺广度骨架（不写深度，不许砍单）

先只产出节点骨架：每个节点 `id`(如 `RAG-01`)/`title`/`deps`(DAG)/`bloom`/`area`，**先不写 objective/seeds/anchors**。

- **节点数量由领域本身决定，不许为了省事或凑整数砍内容。** 一个值得认真学的系统，覆盖全面常需要 **15~40 个**节点；宁可拆细，不许把一大块塞进一个节点。
- 自问并写下"为什么没漏掉任何主要领域"。

## 第 4 步 · 查漏（先把广度补全，再谈深度）

对照第 2 步的调研，逐项检查：**还有哪些重要内容被漏掉、或被一句话合并掉了？** 尤其揪"该单独成节点的硬骨头"。补进骨架。广度没补全之前，不进下一步。

## 第 5 步 · 逐节点独立深挖（核心：杜绝敷衍）

**一个节点一个节点地**写深度，每个都当成独立任务：
- `objective`：学到什么程度，**具体、可验收**（不是教科书定义、不是"理解 XX"这种空话）。
- `socratic_seeds`：**≥2 个**（建议 2~3 个）真能逼出思考的种子问题，不是 yes/no 送分题。
- `grounding_anchors`：**具体、真实存在、可联网核实**的对标对象/论文/文档（别写"相关资料"这种）。
- 确认 `bloom`。
- **深度标准对第 1 个和最后 1 个节点完全一致。** 你若发现自己在后段写得越来越短，停——那就是敷衍信号，回去把它写到和开头一样厚。

## 第 6 步 · 逐节点质检 + 打回重做

每个节点深挖完，**换一个挑剔的视角逐条评**：objective 够不够具体可验收？种子是不是真逼思考？对标是不是真实具体？bloom 配不配？**任一项不过关 → 当场重写这一个节点**（带着批评意见），直到过关。别让任何一个单薄节点蒙混进去。

## 第 7 步 · 查漏补缺到"挤不出新东西"（completeness-until-dry）

站在领域专家角度终审整张课：还有遗漏的主要内容吗？有哪个节点其实该再拆细？依赖顺序合理吗？**有就补/拆，补完再审一遍，直到连续一轮挤不出新东西为止**——不是审一次就完。

## 第 8 步 · 确定性体检（机器兜底，治敷衍的最后一道闸）

把课程写进 `packs/<id>/`，然后跑引擎的两个确定性检查，**有问题就回到第 5~7 步修，循环到干净**：

```bash
python scripts/tutor.py validate --pack <id>   # 结构错(成环/悬空/重复/bloom非法) + 质量警告(空目标/objective过短/种子<2)
python scripts/tutor.py audit    --pack <id>   # 深度体检：前后段是否塌、揪出明显比同伴单薄的『敷衍嫌疑』节点
```

`audit` 会算每个节点的厚度、报"后半段比前半段薄百分之几"、列出 objective 过短/种子<2/无对标的节点。**audit 还在报敷衍嫌疑，就没建完。**

## 第 9 步 · 按用户水平校准 + 活课程

`init` 后对最前面几个节点做入口校准（`assessment_protocol` 第一节），已会的 `master` 跳过、裁短路径。学习过程中发现某节太浅/太跳/缺一环，直接改 `curriculum.json`（加节点、调依赖、补深度），引擎下次 load 自动纳入、重算解锁。

---

## 落地命令小结

```bash
python scripts/tutor.py new-pack <id> "<课程标题>"   # 建空骨架
#  → pack.json: learner_profile/goal/target_depth/orientation/assessment_styles/grounding
#  → curriculum.json: 第3~7步打磨好的节点图（每个节点 objective具体 + ≥2种子 + 真实anchors）
#  → grounding.md: 第2步的对标锚点
python scripts/tutor.py validate --pack <id>   # 结构+质量闸门（有 error 默认拒绝 init）
python scripts/tutor.py audit    --pack <id>   # 深度体检，揪敷衍，循环修到干净
python scripts/tutor.py init --pack <id>
python scripts/tutor.py boot
```

## 自动生成最容易翻车的场景（提前认怂、诚实处理）

- **没有好的公开来源**（小众/保密/太新）：明说"可靠资料有限，部分是推断"，宁可少而准。
- **快速变动的领域**：标注资料时间，多依赖原理少依赖具体数字/榜单，靠复习时联网更新。
- **观点驱动/有争议的领域**：不假装有唯一答案；测验改为"能否站住一种立场并接住反驳"，orientation 设成论证攻防。
- **纯技能/操作类**（编程/乐器）：节点 objective 要含"动手产出物"，测验加"做给我看"。
