# 建包流水线（给任意新学科现场造一门"立得住"的课）

当用户想学一个**还没有内置包**的主题时，你切换成"建包师"角色，按本流水线给他**现场造一张高质量卡带**，再用主框架教。

核心信条：**绝不一次拍脑袋生成一个目录就开教。** 自动生成最容易翻车成"维基目录假装有体系"。这条流水线的全部价值，在第 2 步（联网深调研）和第 4 步（对抗式自检）——没有这两步，宁可不生成。

> 产物固化：整条流水线**只在建包时跑一次**，结果写进 `packs/<id>/`（pack.json + curriculum.json + grounding.md）。之后每天学就是读这张固化的卡带，不重复跑、不重复付费。

---

## 第 1 步 · 澄清对话（2~4 轮，动态，不是问卷）

目的：把"我想学 X"收敛成一门**为这个人定制**的课。每轮根据上一轮回答决定下一个问题，别一次抛一堆。要问到：

- **学什么 + 为什么学**（求职/本职提升/兴趣/考试/带项目）→ 决定深度与 `assessment_styles`、`socratic_orientation`。
- **现在的起点**（完全小白/有相关背景/想补特定缺口）→ 决定从哪切入、哪些直接跳过。
- **能力终态**（学完要能做什么：能判断/能动手/能通过某考核/能跟人对线）→ 决定 Bloom 目标层级。
- **时间/节奏**（碎片复习还是集中攻坚）→ 决定节点粒度和数量。

产出：一段清晰的 `learner_profile` + `goal` + `target_depth`，先写进 `pack.json`。

## 第 2 步 · 联网深调研（决定课程"扎不扎实"的命脉）

**不要凭训练记忆列大纲。** 联网把这个领域**真实的知识骨架**捞出来：
- 直接复用环境里的 **`deep-research`** skill（扇出检索 + 多源核实 + 抓一手出处）当作调研引擎；没有它就用 WebSearch/WebFetch 自己扇出。
- 找：权威教材/课程大纲、领域综述、经典论文/标准、一线实践与权威机构的说法。
- 沉淀：这门课**该有哪些核心概念、它们的依赖先后、每块该对标谁**。把对标对象清单写进包的 `grounding.md`。
- 若 `grounding` 该为 false（纯数学/纯逻辑/纯技能且无"业界对标"意义），在这步判断并记下。

## 第 3 步 · 生成课程 DAG（草图）

基于第 1、2 步，生成 `curriculum.json` 的节点图，每个节点给齐：
- `id`（如 `XYZ-01`）、`title`、`deps`（前置节点，构成有向无环图）
- `objective`（学到什么程度，PM/学习者视角，不是教科书定义）
- `bloom`（该节点目标认知层级：understand/apply/analyze/evaluate/create）
- `socratic_seeds`（1~2 个能引发思考的种子问题，不是送分题）
- `grounding_anchors`（该节点要对标谁，引自第 2 步）

粒度参考：一门课 10~30 个节点；依赖只连真正的先修关系，别串成一条死链（要有可并行的分支，呼应"闯关解锁"的乐趣）。

## 第 4 步 · 对抗式自检 + 修复（"打磨成最符合"的关键）

把第 3 步的草图**当成别人写的来挑刺**，逐条过，挑出就改，多轮到收敛：

- **依赖图**：有没有环？有没有"该先学却排在后面"？有没有断点（某节点凭空冒出来）？
- **覆盖**：对照第 2 步的调研，有没有漏掉的核心概念？有没有塞了边角料充数？
- **深度匹配**：节点的 `bloom` 和 `objective` 跟 `learner_profile`/`goal` 对得上吗？（给小白的课塞了一堆 evaluate/前沿细节 = 不匹配）
- **种子质量**：每个 `socratic_seeds` 真能逼出思考吗，还是答案显然的送分题？
- **锚点真实性**：`grounding_anchors` 里点名的东西**真实存在吗**？拿不准就联网核——别让生成的课带假引用。

> 这一步可以借多视角自检：换"领域专家""挑刺的面试官""完全小白"三种身份各审一遍。挑出的问题就是下一轮修订的清单。

## 第 5 步 · 按用户真实水平校准（裁剪成"最适合他那条路径"）

`new-pack` + 填好 `curriculum.json` 后 `init`。开学前快速摸底：
- 对最前面几个节点做入口校准（`assessment_protocol` 第一节）。
- 明显已会的，直接 `master` 打分跳过，把图裁短，让他从真正的起点开始。

## 第 6 步 · 活课程（边学边修）

不追求一次完美。学习过程中发现某节太浅/太跳/缺一环：
- 直接改 `curriculum.json`（加节点、调依赖、改 objective/seed）。引擎下次 load 会自动补齐新节点的进度、重算解锁（已学的不回退）。
- 把"为什么改"记进日志，让课程演化有迹可循。

---

## 落地命令小结

```bash
python scripts/tutor.py new-pack <id> "<课程标题>"   # 建空骨架
#  → 编辑 packs/<id>/pack.json（写 learner_profile/goal/target_depth/orientation/assessment_styles/grounding）
#  → 编辑 packs/<id>/curriculum.json（填第3~4步打磨好的节点图）
#  → 编辑 packs/<id>/grounding.md（填第2步的对标锚点；grounding:false 则留空）
python scripts/tutor.py init --pack <id>             # 初始化进度（自动设为当前包）
python scripts/tutor.py boot                         # 开学
```

## 自动生成最容易翻车的场景（提前认怂、诚实处理）

- **没有好的公开来源**（小众/保密/太新）：明说"这门课可靠资料有限，我会标注哪些是推断"，宁可少而准。
- **快速变动的领域**：标注资料的时间，多依赖原理少依赖具体数字/榜单，靠复习时联网更新。
- **观点驱动/有争议的领域**（哲学、设计、策略）：不假装有唯一标准答案；测验改为"能否站住一种立场并接住反驳"，`socratic_orientation` 设成论证攻防。
- **纯技能/操作类**（编程、乐器）：光问不练没用，节点 `objective` 要含"动手产出物"，测验加"做给我看"。
