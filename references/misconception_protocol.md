# 误概念账本协议（把"似懂非懂"变成可清的待办）

学习里最贵的不是"没学过"，是**学错了还不自知**——错误的心智模型不主动打掉，会在面试、工作、新场景里反复犯。本协议把这些断层做成一个**结构化、可闭环**的账本（升级自原版的流水账式 gaps）。

## 一、什么算一条误概念（vs 普通没学到）

记 `miss` 的时机不是"他还没学"，而是**他有一个错的/混乱的理解**：
- 把 A 和 B 搞混（"他以为 MIG 和 MPS 一回事"）。
- 因果搞反、动机说错（"他以为 gang scheduling 是为了省钱，其实是防资源死锁浪费"）。
- 似懂非懂、一追就露怯的断层（连环追问刨到他答不上来的那个点）。
- 自信却答错的点（校准危险区里的）。

普通的"这块还没学/还没到"不必记 miss——那是进度，引擎已经管了。

## 二、闭环：open → 复测 → retired

账本是 `state/<pack>/misconceptions.json`，引擎维护，命令：

```bash
echo "把A和B搞混 / 因果说反了：具体描述" | python scripts/tutor.py miss add <节点ID>   # 暴露时立刻记
python scripts/tutor.py miss list --open-only                                          # 看还没清的
python scripts/tutor.py miss retest <Mxxx>                                             # 复测了一次（还没纠正）
python scripts/tutor.py miss resolve <Mxxx>                                            # 确认纠正了 → retired
```

- **add**：测验/追问里一暴露就记，别等。描述要具体到"错在哪"，不是"XXX没掌握"。
- **boot 优先拎**：每次开机，引擎把 open 的误概念拎到最前，提醒复习时优先打。
- **复测**：复习该节点时，专门设计一道题打这个误概念。没纠正过来就 `retest`（计数+1，留着）。
- **resolve**：用户**独立地**把它讲对了（不是你提示出来的），才 `miss resolve` 退役。

## 三、纪律

- **不许烂尾**：一条 open 误概念别无限期挂着。每条最迟在它所属节点的下一次复习时复测一次。
- **纠正要靠他自己讲对**，不是你纠正完他点头。点头不算 retired。
- **跨节点的混淆**记在更相关的那个节点上，复测时把两个概念放一起逼他辨别（呼应 `review_protocol` 的交错）。
- 误概念**单独走 miss**，别混进日志正文——日志是"学了什么"，账本是"还欠什么"，分开才好清。
